# Физическая модель данных M8 v1.0

[Executable Baseline](../index.md) | [Data Baseline](index.md)

## Правила

- каждый сервис владеет собственным префиксом таблиц;
- запись агрегата, Operation и Outbox выполняется в одной локальной транзакции;
- внешние сервисы не читают таблицы владельца;
- проекции являются удаляемыми и восстанавливаются повторным проигрыванием событий;
- optimistic locking использует `revision` и `etag`;
- secret material не помещается в `attributes`, Outbox или audit payload;
- персональные данные хранятся только в контексте-владельце.

## Общие таблицы

`outbox`, `inbox`, `idempotency_keys` и `operations` создаются для каждого сервиса.
Доменные таблицы создаются из утверждённого Data Ownership Registry.

## Следующий уровень детализации

Перед реализацией конкретного Feature Prompt общая колонка `attributes`
заменяется типизированными колонками и отдельными дочерними таблицами там,
где это необходимо для инвариантов, индексов или эффективных запросов.
